# M5 — Trip Management Module

**Priority:** P0 | **Phase:** 1 | **Complexity:** Medium

## Description
Complete trip CRUD: create, edit, delete, duplicate, archive, restore, save draft, and publish. Trip metadata includes title, description, cover image, dates, traveler count, budget target, trip type, privacy, transport/accommodation preferences, origin city, and currency.

## Acceptance Criteria
- [ ] Create trip form with all metadata fields
- [ ] Edit trip form (pre-populated)
- [ ] Delete trip with confirmation dialog
- [ ] Duplicate trip (deep copy with stops and activities)
- [ ] Archive/restore trip
- [ ] Draft save (auto-save every 30 seconds)
- [ ] Trip status management (draft → active → completed → archived)
- [ ] Privacy control (private, shared, public)
- [ ] Cover image upload to Supabase Storage
- [ ] My Trips page with filtering (all, active, draft, archived)
- [ ] Trip detail page (overview with links to itinerary, budget, packing, notes)

## Database Touchpoints
- `trips` table (full CRUD)
- `trip_stops`, `trip_activities`, `budgets`, `packing_items`, `notes` (cascade on delete, deep copy on duplicate)
- Supabase Storage `trip-covers` bucket

## API Touchpoints
- GET /api/v1/trips
- POST /api/v1/trips
- GET /api/v1/trips/:id
- PATCH /api/v1/trips/:id
- DELETE /api/v1/trips/:id
- POST /api/v1/trips/:id/duplicate
- POST /api/v1/trips/:id/archive

## Dependencies
- M1 (Foundation), M2 (Authentication)
