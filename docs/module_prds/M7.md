# M7 — Destination & Activity Search Module

**Priority:** P0 | **Phase:** 1 | **Complexity:** Medium

## Description
Destination discovery and activity lookup with filtering, sorting, and quick-add to trips. Uses internal database seeded with destination/activity data.

## Acceptance Criteria
- [ ] Destination search by city/country name
- [ ] Destination discovery page with trending/popular destinations
- [ ] Destination cards with budget index, highlights, tags
- [ ] Budget and region filters for destinations
- [ ] Activity search by name, category, destination
- [ ] Activity filtering (price, duration, popularity, family-friendly, indoor/outdoor, free/premium)
- [ ] Activity sorting (price, rating, duration)
- [ ] Quick-add activity to trip (select trip → select stop → add)
- [ ] Search results pagination
- [ ] Empty state and loading skeletons

## Database Touchpoints
- `activities` table (read + search)
- Custom `destinations` seed data (JSONB or separate table)

## API Touchpoints
- GET /api/v1/search/destinations
- GET /api/v1/search/activities

## Dependencies
- M1 (Foundation), M2 (Auth), M5 (Trip Management)
