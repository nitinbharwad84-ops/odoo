# M3 — User Profile Module

**Priority:** P0 | **Phase:** 1 | **Complexity:** Low

## Description
Profile viewing, editing, avatar upload, travel preferences selection, and account settings management.

## Acceptance Criteria
- [ ] Profile view page showing user info, avatar, travel preferences
- [ ] Profile edit form (first_name, last_name, phone, city, country, language)
- [ ] Avatar upload to Supabase Storage with preview
- [ ] Travel style selection (backpacker, family, luxury, adventure, foodie, digital_nomad, romantic, business)
- [ ] Notification preferences toggle
- [ ] Account deletion flow

## Database Touchpoints
- `profiles` table (read/write)
- Supabase Storage `avatars` bucket

## API Touchpoints
- GET /api/v1/profile
- PATCH /api/v1/profile
- POST /api/v1/profile/avatar

## Dependencies
- M1 (Foundation), M2 (Authentication)
