# M2 — Authentication Module

**Priority:** P0 (Must Ship) | **Phase:** 1 | **Complexity:** Medium

## Description
Implement complete authentication system using Supabase Auth: email/password signup, login, logout, email verification, forgot password, reset password, session persistence, and auth middleware.

## Acceptance Criteria
- [ ] Signup form with validation (first_name, last_name, email, password, confirm_password)
- [ ] Login form with email/password
- [ ] Logout functionality with session cleanup
- [ ] Forgot password flow (email with reset link)
- [ ] Reset password page with token validation
- [ ] Email verification flow
- [ ] Session persistence across page reloads
- [ ] Auth middleware redirecting unauthenticated users
- [ ] Protected API routes requiring valid session
- [ ] Error handling for all auth states
- [ ] Loading states and form validation feedback

## Database Touchpoints
- `users` table (read/write via Supabase Auth)
- `profiles` table (create on signup)

## API Touchpoints
- POST /api/v1/auth/register
- POST /api/v1/auth/login
- POST /api/v1/auth/logout
- POST /api/v1/auth/forgot-password
- POST /api/v1/auth/reset-password
- GET /api/v1/auth/me
