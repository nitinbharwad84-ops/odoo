# M11 — Collaboration Module

**Priority:** P1 | **Phase:** 1 | **Complexity:** Medium

## Description
Invite collaborators to trips with role-based access (owner, editor, viewer). Shared editing of itineraries, notes, and checklists.

## Acceptance Criteria
- [ ] Invite collaborators by email
- [ ] Role assignment (editor/viewer)
- [ ] Invitation acceptance/decline flow
- [ ] Collaborator list management
- [ ] Remove collaborator
- [ ] Change collaborator role
- [ ] Editors can: edit itinerary, modify activities, update checklist, add notes
- [ ] Viewers can: view trip, read notes, comment (future)
- [ ] Owner can: full control + collaborator management
- [ ] In-app notification on invite

## Database Touchpoints
- `collaborators` table (CRUD)
- `notifications` table (write invite notifications)

## API Touchpoints
- POST /api/v1/trips/:id/invite
- GET /api/v1/trips/:id/collaborators
- PATCH/DELETE /api/v1/collaborators/:id

## Dependencies
- M1, M2, M5, M14 (Notifications)
